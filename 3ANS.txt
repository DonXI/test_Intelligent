3. หากท่านได้รับมอบหมายให้รับหน้าที่พัฒนาระบบ POS คิดเงินหน้าร้านเกี่ยวกับระบบ Fitness และ
มีการประเมินจาก Developer lead หรือ Project Manager ว่าต้องใช้เวลาพัฒนาทังหมด 120 
วัน และงานในส่วนที่ต้อง Develop กว่า 50 Function ย่อย
ใน Project นี้ที่ท่านรับผิดชอบทางทีมไม่ได้กําหนดเวลาเข้าออกงาน หรือต้องเข้ามาทํางานที่บริษัทฯ 
แต่เน้นผลลัพธ์ของงานเป็นสําคัญ ท่านจะออกแบบวิธีการทํางานของท่านอย่างไร 
จะใช้วิธีรายงานความคืบหน้างานอย่างไร ทําอย่างไรเมื่อติดปัญหา และจะจัดการปัญหาอย่างไรหากเล็งเห็นว่างาเริ่มช้ากว่าปกติ 
จงอธิบายให้ครบถ้วนแต่ละประเด็น

3.1 ท่านจะออกแบบวิธีการทํางานของท่านอย่างไร 
ระบุเป้าหมายงานของแต่ละเดือน(30วัน) 
ในเดือน1-3เดือนแรก แบ่งทำ function ที่สำคัญๆก่อน ดูจากการเชื่อมต่อไปยังส่วนอื่นที่ต้องใช้งานเป็นส่วนมาจากส่วนนี้
ซึ่ง function หลักๆต้องเสร็จ โดยอาจจะมีการมา update ผลการทำงานหรือบอกปัญหาที่เจอผ่านการคุยกันทุกสัปดาห์ 
และช่วยกันคิดหรือเสนอแนวแทงการแก้ปัญหา การพูดคุยกันจะเลือกวันเวลาที่ทุกคนว่างตรงกัน 
ในเดือนที่ 1-2 เดือนสุดท้ายจะมีการทดสอบระบบ และปรับแก้ระบบในส่วนต่างๆ แสดงผลให้ดูในที่ประชุมเพื่อพูดคุยกัน

3.2 จะใช้วิธีรายงานความคืบหน้างานอย่างไร 
นัดคุยกันสัปดาห์ละ1-2ครั้ง นัดประชุมกันภายในกลุ่ม แสดงผลการทำงาน หรือ demo ให้ดู

3.3 ทําอย่างไรเมื่อติดปัญหา และจะจัดการปัญหาอย่างไรหากเล็งเห็นว่างานเริ่มช้ากว่าปกติ 
สอบถามพี่ๆในทีม เพื่อแนะแนวทางการแก้ปัญหา หากงานช้ากว่าปกติก็จะมีการเร่งตัวเองและพยายามให้ทำงานให้เสร็จทันเวลา 
และบอกปัญหาที่เกิดขึ้นกับหัวหน้างาน
